介绍一下 trivial type 和 POD type

trivial type

trivial类型在内存中连续存放，trivial类型对象可以拷贝到一个char数组，然后再安全地拷贝生成另一个trivial类型对象。(需要注意的是， 由于字节对齐需要，对象里的成员之间可能会存在padding bytes)
trivial类型必须包含默认 构造函数、默认 拷贝构造函数、默认 拷贝赋值函数、默认 析构函数，另外还需要满足：

  • 没有虚函数
  • 如果有基类，基类必须是trivial类型
struct Foo {
Foo() = default;// -> trivially constructible
~Foo() = default; // -> trivially destructible
Foo(Foo const &) = default;// -> trivially copy constructible
auto operator=(Foo const &) -> Foo & = default; // -> trivially copy assignable
Foo(Foo&&) = default; // -> trivially move constructible
auto operator=(Foo&&) -> Foo& = default; // -> trivially move assignable
};

POD type

POD(Plain Old Data)类型要求比trivial类型更严格，简单来说就是去除一切C++功能，纯C类型就对了。
POD数据显然也是内存连续存放的。
